## StudentCollection

В работе реализован контейнерный класс StudentCollection, который хранит и управляет объектами класса Student (из ЛР-1).

Коллекция позволяет:
- хранить множество студентов
- управлять добавлением и удалением
- выполнять поиск и фильтрацию
- сортировать объекты
- работать с коллекцией как с обычным списком

## Атрибуты класса

_items — список объектов Student, хранящихся в коллекции

## Основные методы

### Управление коллекцией

add(student) — добавить студента (с проверкой типа и дубликатов)
remove(student) — удалить студента
remove_at(index) — удалить студента по индексу
clear() — очистить коллекцию
is_empty() — проверить, пуста ли коллекция

### Получение данных

get_all() — вернуть список всех студентов
__len__() — количество студентов (len(collection))
__iter__() — итерация (for student in collection)
__getitem__(index) — доступ по индексу (collection[0])

### Поиск

find_by_fio(fio) — поиск по ФИО
find_by_group(group) — поиск по группе
find_by_fio_contains(substring) — поиск по части ФИО
find_active() — найти активных студентов

### Сортировка

sort(key, reverse=False) — универсальная сортировка
sort_by_fio() — по ФИО
sort_by_group() — по группе
sort_by_gpa() — по среднему баллу
sort_by_course() — по курсу
sort_by_age() — по возрасту

### Фильтрация (логические операции)

Методы возвращают новую коллекцию:
get_active_students() — только активные студенты
get_graduated_students() — только выпущенные студенты
get_expelled_students() — только отчисленные студенты
get_students_by_gpa_range(min, max) — по диапазону GPA
get_students_by_course(course) — по курсу
get_students_by_group(group) — по группе

### Магические методы

__str__ — красивый вывод коллекции
__repr__ — техническое представление

## Ограничения

- можно добавлять только объекты Student
- нельзя добавлять дубликаты (по ФИО и дате рождения)
- индексация с проверкой границ
- строгая типизация аргументов
