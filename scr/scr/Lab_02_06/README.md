## TypedCollection
В работе реализован Generic-класс TypedCollection[T], который может хранить элементы любого типа. 
В отличие от обычной коллекции из ЛР-2, этот класс знает, какой тип объектов в нём хранится, и обеспечивает типобезопасность на уровне аннотаций.

#### Коллекция позволяет:

- хранить элементы любого типа (благодаря TypeVar T)

- управлять добавлением и удалением

- выполнять поиск, фильтрацию и преобразование через стратегии (делегаты)

- сортировать объекты по переданной функции-ключу

- работать с коллекцией как с обычным списком (через магические методы)

## TypeVar и Generic
В работе используются следующие TypeVar:

T = TypeVar('T') # Без ограничений — любой тип
R = TypeVar('R') # Для метода map (тип результата может отличаться)
D = TypeVar('D', bound=Displayable) # Только объекты с методом display()
S = TypeVar('S', bound=Scorable) # Только объекты с методом score()

### Generic-класс TypedCollection[T] параметризуется типом T. Это позволяет создать одну универсальную коллекцию для любых типов данных (Student, str, int и т.д.), сохраняя при этом типобезопасность.

## Protocols (структурная типизация)
Protocol — это способ описания интерфейса без явного наследования. Если объект имеет нужные методы — он соответствует протоколу (утиная типизация).

## Optional
Optional[T] — это тип, который может быть либо T, либо None. Используется в методе find():

## Основные методы TypedCollection
Управление коллекцией

- add(item) — добавить элемент (с проверкой типа через аннотации)

- remove(item) — удалить элемент

- remove_at(index) — удалить элемент по индексу

- clear() — очистить коллекцию
