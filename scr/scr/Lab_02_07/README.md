#  Лабораторная работа: Консольное приложение
## Цель работы
### Объединить все знания, полученные в ЛР1–ЛР6, в единое работающее приложение.
### Реализовать интерактивный CLI-интерфейс с меню и вводом пользователя.


### Трёхслойная архитектура
#### Приложение разделено на три независимых слоя:

- cli.py – слой представления (только ввод/вывод, меню, форматирование)

- app.py – слой бизнес-логики (операции над коллекцией, валидация)

- storage.py – слой данных (сохранение и загрузка в JSON)

#### Такой подход обеспечивает:

разделение ответственности

возможность замены интерфейса (например, на GUI)

переиспользование бизнес-логики

упрощение тестирования и поддержки

### Основные компоненты (app.py)
StudentApp – класс бизнес-логики, предоставляющий методы для работы с коллекцией

- add_student()	Добавление студента (с валидацией)
- remove_student()	Удаление студента по индексу
- clear_collection()	Очистка коллекции
- find_by_fio()	Поиск по ФИО
- find_by_group()	Поиск по группе
- filter_by_excellent()	Фильтр отличников
- filter_by_gpa_range()	Фильтр по диапазону GPA
- filter_by_course()	Фильтр по курсу
- filter_by_active()	Фильтр активных студентов
- sort_by_fio()	Сортировка по ФИО
- sort_by_gpa()	Сортировка по GPA
- sort_by_course()	Сортировка по курсу
- sort_by_age()	Сортировка по возрасту
- sort_by_group()	Сортировка по группе
- get_table_string()	Форматированная таблица студентов

### Консольное меню с циклом
После каждого действия снова показывать меню, обрабатывать некорректный ввод.

### Сохранение и загрузка в JSON?

При запуске – load() читает файл, восстанавливает объекты

При выходе – save() сохраняет текущее состояние

Методы student_to_dict() и dict_to_student() конвертируют объекты
