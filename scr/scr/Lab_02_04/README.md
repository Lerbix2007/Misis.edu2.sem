## Интерфейсы (контракты)
- В работе созданы два интерфейса (абстрактных класса):

### Printable
Интерфейс, который обязывает объект уметь предоставлять строковое представление.
Абстрактный метод:
to_string() -> str – возвращает строку с информацией об объекте.

### Comparable
Интерфейс, который обязывает объект уметь сравнивать себя с другим объектом.
Абстрактный метод:
compare_to(other: Comparable) -> int – возвращает:

-1, если текущий объект «меньше» другого,

0, если объекты равны,

1, если текущий объект «больше» другого.

### Классы, реализующие интерфейсы
- Student (базовый класс)
Реализует оба интерфейса:

to_string() → возвращает строку вида: "Студент: ФИО | Группа | GPA: ..."

compare_to(other) → сравнивает сначала по GPA, затем по ФИО.

- BachelorStudent (наследник Student)
Переопределяет методы интерфейсов:

to_string() → добавляет информацию о курсе.

compare_to(other) → сравнивает по GPA, затем по курсу, затем по ФИО.

- MasterStudent (наследник Student)
Переопределяет методы интерфейсов:

to_string() → добавляет тему исследования и количество публикаций.

compare_to(other) → сравнивает по GPA, затем по количеству публикаций, затем по ФИО.

### Универсальные функции, работающие через интерфейсы
print_all(items: list[Printable])
Принимает список любых объектов, реализующих интерфейс Printable, и выводит их строковое представление.

find_max(items: list[Comparable]) -> Comparable
Принимает список объектов, реализующих интерфейс Comparable, и возвращает максимальный элемент согласно логике compare_to().
